# examples（オプション）

各 example は独立した Markdown ファイルで、実際にテストされた 1 組の input / output を記録します。

## 例の追加

`example_01.md` をコピーし、リネーム（`example_02.md`、`example_03.md`……）して、新しいシナリオを記入します。

## 推奨する数

| Skill の複雑度 | 推奨する例の数 |
|-----------------|-------------------|
| 単一の input / output | 1 |
| 複数の input シナリオ | 2–3 |
| スクリプトまたはマルチターン対話を含む | 3 以上 |
